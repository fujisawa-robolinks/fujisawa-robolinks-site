# 技術選定メモ（運用方針反映版）

## 前提

- GitHub Pages で公開
- 活動更新は Instagram を中心に運用
- ブログは大きな大会記事のみ
- 多言語化と大会専用ページは当面対象外
- 写真投稿が増える想定

## 結論

公開前に Astro 導入を推奨。

## Astro を選ぶ理由

- 静的サイトとして GitHub Pages と相性が良い
- ページが増えても共通レイアウト管理がしやすい
- 画像最適化や構造化を段階的に追加しやすい
- 将来要件が増えても、素のHTMLより拡張しやすい

## コンテンツ運用の整理

- 活動内容: Instagram へのリンク・埋め込み中心
- ブログ: 大会レポートのみ掲載（頻度低め）
- アルバム: 写真を中心に掲載（最も更新頻度が高い）

## 非対象（スコープ外）

- 多言語化
- 大会ごとの個別特設ページ
- CMS導入（まずは手動運用）

## VS Code 開発環境の推奨

- Astro 拡張を導入して Astro ファイルを快適に編集
- Prettier でフォーマット統一
- 開発時は `npm run dev` でライブプレビュー
